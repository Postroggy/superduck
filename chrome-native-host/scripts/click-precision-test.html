SuperDuck 点击精度测试页

只验证左键单击。正确做法不是只做一块“密密麻麻的小按钮”，而是分层测：先测大按钮校准坐标，再测中等密度，再测极限密度和贴边误点。这个页面已经把这些场景拆开了，点完会立即记录目标、坐标、距目标中心偏移和离边缘余量。

建议浏览器缩放保持 100% 先测大按钮，再测 24px / 16px / 12px 网格 空白区域点击会记为 miss 推荐窗口最大化后再跑 CLI

1. 基线校准

先确认你的插件或 CLI 在正常尺寸目标上没有整体偏移。若这里都不稳定，后面的密集按钮测试没有参考价值。

2. 密集网格

从 24px 到 12px 逐级变难。每个格子都有唯一 ID，适合验证“目标点位正确，但是否误点到邻居”。

3. 相邻误点陷阱

这一组专门看“靠得很近时会不会点到隔壁”。横向条带适合测 X 轴偏差，纵向条带适合测 Y 轴偏差。

横向条带 14 个目标，宽 18px，高 30px，间隙 1px
纵向条带 14 个目标，宽 30px，高 18px，间隙 1px

4. 框边靶位

这一块用于看贴边位置是否存在额外偏移。它不是浏览器真实 viewport 边缘，但能很好暴露“目标靠边时更容易点错”的情况。

边缘靶位区域
8 个 28px 靶点，覆盖四角和四边中点。
